# 面试雷达 UI 风格方向

状态：待审查  
日期：2026-06-20

## 1. 当前结论

UI 改为二次元风，但产品本质仍是面试学习工作台。

方向不是游戏官网、抽卡页、角色官网，也不是满屏插画。第一版要做“二次元学习工作台”：角色陪伴 + 任务板 + 练习反馈 + 题库管理。

未登录时进入登录/注册页面；登录后第一屏进入 Dashboard。

## 2. 推荐风格：二次元学习工作台

关键词：

- 轻番剧感
- 学习任务板
- 角色陪伴
- 清爽高可读
- 克制动效

用户打开后应该像进入一个“面试训练社团活动室”：今天的练习任务、薄弱点、候选题、角色提示都在，但核心操作还是生成题、练题、看反馈。

## 3. 视觉系统

推荐浅色二次元，不做默认深色。

```css
:root {
  --bg: oklch(0.99 0.01 250);
  --surface: oklch(1 0 0);
  --surface-tint: oklch(0.96 0.025 250);
  --ink: oklch(0.24 0.03 260);
  --muted: oklch(0.48 0.035 260);
  --primary: oklch(0.62 0.16 255);
  --primary-strong: oklch(0.48 0.18 258);
  --accent: oklch(0.78 0.16 345);
  --mint: oklch(0.78 0.12 170);
  --warning: oklch(0.78 0.15 82);
  --danger: oklch(0.62 0.17 25);
  --border: oklch(0.9 0.018 260);
}
```

用法：

- 主色偏蓝紫，用于主按钮、当前导航、重点进度。
- 粉色只做点缀，用于角色提示、鼓励、徽章。
- 薄荷绿用于掌握、完成、通过。
- 黄色用于待复习和提醒。
- 背景保持清爽，不能铺满高饱和渐变。

## 4. 布局方向

桌面端：

- 左侧导航：Dashboard、AI 生成、题库、练习、资料、设置。
- 顶部目标条：当前面试目标、倒计时、AI 接入未连接状态。
- Dashboard 做成“今日训练面板”：今日任务、薄弱点、最近反馈、角色提示。
- 右侧可以有一个小型“训练搭档”区域，放角色立绘或头像、状态提示、鼓励语。
- 练习页保持专注，角色只在侧栏给提示，不遮挡题干和输入区。

移动端：

- 底部导航。
- 角色提示折叠为顶部小气泡。
- 练习页优先显示题干和回答输入，反馈单独往下展开。

## 5. 角色与二次元元素

第一版可以有角色感，但不做复杂角色系统。

允许：

- 一个默认训练搭档头像或半身视觉位。
- 角色气泡提示今日任务、回答建议、复习提醒。
- 少量贴纸感标签，例如“今日训练”“薄弱警报”“已掌握”。
- 登录页可以更有二次元氛围。

不做：

- 抽卡。
- 角色养成。
- 商店。
- 皮肤。
- 多角色系统。
- 大量剧情文本。
- 满屏粒子和高频动画。

## 6. 组件风格

建议：

- 卡片圆角 12px，不要过圆。
- 按钮可更活泼，但仍使用明确动词：生成候选题、确认入库、开始练习、查看反馈。
- 题库列表仍以可扫视为主，不做全卡片画廊。
- 状态标签可以更像二次元 UI 徽章，但文字必须清晰。
- 空状态用角色短提示，不写长说明。
- 动效 150 到 220ms，只用于状态变化、完成反馈、角色提示出现。

避免：

- 全站紫粉渐变。
- 低对比浅粉文字。
- 装饰压过信息。
- 每个页面都放大立绘。
- 游戏化过度。
- 日系标题字体用于正文和表格。
- 登录后还出现营销 hero。

## 7. 页面风格

登录/注册：

- 可比工作台更有氛围。
- 参考 [animatedlogin](https://github.com/guohaolian/animatedlogin/blob/main/README.md) 的左右分栏：左侧互动角色，右侧登录表单。
- 角色可响应鼠标、输入焦点、密码输入、登录失败和按钮悬停。
- 密码输入时角色回避或遮眼；登录失败时角色短暂沮丧或摇头。
- 表单仍要清楚、可读、可键盘操作。
- 移动端改为上下结构，角色缩成头像或小气泡，不挤压表单。

Dashboard：

- 像“今日训练作战板”。
- 重点是下一题、薄弱点、最近反馈、候选题组。
- 角色提示只做辅助，不占主视觉中心。

AI 生成：

- 像“出题委托单”。
- 支持岗位目标、资料增强、补薄弱点三种模式。
- 候选题确认区要像批量审核列表，不像聊天消息流。

练习：

- 像“模拟问答训练室”。
- 题干、回答区、参考答案、AI 反馈必须清楚。
- 角色只给轻提示，不替代反馈报告。

资料：

- 像“资料收纳箱”。
- 简历、JD、项目笔记、复习资料按类型归档。

设置：

- 保持简单。
- AI 接入只显示配置原型和未连接状态，不固定地址。

## 8. 参考产品取舍

- OpenExam 借鉴学习闭环，不借鉴考试试卷中心。
- Yoodli 借鉴练习反馈，不借鉴企业培训气质。
- Final Round AI 借鉴简历/JD 个性化，不做实时隐形辅助。
- animatedlogin 借鉴登录页互动角色和表单反馈，不把它扩成全站动画。
- 二次元参考只取视觉语气：角色陪伴、任务板、状态徽章，不取抽卡和商业化玩法。

## 9. UI 审查标准

新的 UI 写完后，用这些标准判断：

- 是否一眼是二次元风，但仍像学习工具。
- 是否登录后直接进入工作台，不是营销页。
- 是否能看清今天该练什么。
- 是否能理解题目来自岗位目标或资料。
- 练习页是否坚持先回答再看答案。
- 角色是否辅助学习，而不是抢任务。
- 移动端是否能完成一次练习。
- 页面不出现支付、充值、套餐。

## 10. 第一版 UI 默认值

为避免开工后反复改方向，第一版默认：

- 采用二次元学习工作台。
- 做登录、注册、找回密码页面；登录后首屏是 Dashboard。
- 登录页使用 animatedlogin 式互动角色，但实现为 Next.js 客户端小组件 + CSS，默认不引入动画库。
- 一个默认训练搭档，不做角色系统。
- 浅色主题，不做默认深色。
- 不做抽卡、养成、商店、皮肤。
- 不做独立 AI 补题页，AI 生成页内用模式切换。
- AI 接入只在设置页显示配置原型和未连接状态，不固定地址。
- 页面密度按桌面工具设计，移动端只重排，不删核心流程。
